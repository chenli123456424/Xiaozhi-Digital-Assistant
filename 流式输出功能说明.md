【流式输出功能实现说明】

功能目标: 实现 AI 回复流式输出（像"打字"一样逐字显示）

================================================================================
1. 工作原理
================================================================================

【传统模式（已弃用）】
用户输入 → 等待 → 后端完整回答 → 前端显示

【流式模式（新增）】
用户输入 → 后端逐字返回数据 → 前端实时显示 → 用户体验更好

================================================================================
2. 后端实现
================================================================================

文件修改: backend/llm_wrapper.py

新增方法:
- chat_stream(user_message) -> Generator[str]
  作用: 流式调用 LLM，逐段返回回答
  原理: 
    1. 创建 Tongyi LLM 实例，设置 streaming=True
    2. 使用 stream() 方法逐块获取数据
    3. 通过 yield 逐块返回数据

- chat_stream_with_system_prompt(user_message, system_prompt) -> Generator[str]
  作用: 带系统提示词的流式调用
  
实现关键:
```python
streaming_llm = Tongyi(
    model=self.model,
    dashscope_api_key=self.api_key,
    streaming=True,  # 启用流式输出
)

for chunk in streaming_llm.stream(messages):
    if chunk:
        yield chunk  # 逐块返回
```

3. 后端 API 变化
================================================================================

【新增端点】
POST /chat/stream

请求格式:
{
  "message": "你好",
  "conversation_id": "optional"
}

响应格式: Server-Sent Events (SSE)
数据流格式:
data: {"chunk": "你好"}
data: {"chunk": "，"}

...
data: {"done": true, "total": "完整回答"}

【原有端点保留】
POST /chat (非流式，仍然可用)

4. 前端实现
================================================================================

文件修改: frontend/src/components/ChatWindow.jsx

原理:
1. 使用 Fetch API 而不是 Axios（支持流式读取）
2. 调用 /api/chat/stream 端点
3. 使用 Response.body.getReader() 读取流
4. 实时更新 React 状态，触发组件重新渲染

核心代码:
```javascript
const response = await fetch('/api/chat/stream', {
  method: 'POST',
  body: JSON.stringify({ message })
})

const reader = response.body.getReader()
const decoder = new TextDecoder()

while (true) {
  const { done, value } = await reader.read()
  if (done) break
  
  const chunk = decoder.decode(value)
  // 解析 SSE 数据
  // 更新 React 状态，显示流式文本
}
```

【用户体验】
1. 输入消息 → 点击发送
2. 用户消息立即显示
3. AI 消息气泡出现，并逐字"打字"显示
4. 输入框恢复可用状态

5. 关键技术概念
================================================================================

【Server-Sent Events (SSE)】
- 单向通信协议
- 服务器主动推送数据给客户端
- 格式: data: {...}\n\n
- 适用场景: 实时推送、消息流

【Generator（生成器）】
Python 的流式处理机制
- 使用 yield 关键字返回数据
- 暂停执行，等待下一个 next() 调用
- 内存效率高（不需要一次性加载全部数据）

【Fetch API】
现代的 HTTP 通信方式
- 支持流式读取响应
- 使用 response.body.getReader()
- 比 Axios 更灵活

7. 使用对比
================================================================================

【调用 POST /chat（非流式）】
使用场景: 对实时性要求不高的场景
优点: 简单，兼容性好
缺点: 等待时间长，用户体验差

【调用 POST /chat/stream（流式）】
使用场景: 需要实时显示的场景
优点: 用户体验好，逐字显示
缺点: 实现复杂，需要处理 SSE

8. 故障排查
================================================================================

【问题 1: 流式数据不显示】
解决:
1. 检查浏览器控制台是否有错误
2. 验证后端 /chat/stream 端点是否启动
3. 检查 LLM 是否返回数据

【问题 2: 数据乱码】
原因: JSON 转义问题
解决: 检查前端的解析逻辑

【问题 3: 连接断开】
原因: 网络超时或服务器异常
解决: 增加超时时间或重试机制

11. 部署注意事项
================================================================================

需要修改的地方:

backend/main.py:
allow_origins=["*"]  # 改为具体的域名列表

example:
allow_origins=["https://yourdomain.com", "https://www.yourdomain.com"]

【Nginx 代理配置】
如果使用 Nginx 反向代理，需要配置:

location /api/chat/stream {
    proxy_pass http://backend:8000;
    proxy_buffering off;           # 关闭缓冲，实现流式
    proxy_cache off;               # 关闭缓存
    proxy_set_header Connection "";
    chunked_transfer_encoding on;
}
